### 1. CPU(Central Processing Unit，中央处理器) 常见寄存器

#### 这题考什么

考你识别各寄存器的作用，常见于选择题。

#### 这是什么

你可以把 CPU 想成一个正在处理任务的小工厂，寄存器就是工厂里贴身放东西的几个小盒子。

- 有的盒子放“下一步去哪儿”，比如 `PC`
- 有的盒子放“手里正在干什么”，比如 `IR`
- 有的盒子放“要去哪个仓库拿货”，比如 `MAR`
- 有的盒子放“刚拿到或正准备写回的数据”，比如 `MDR`

#### 最小概念

- `PC(Program Counter，程序计数器)`：下一条指令地址。
- `IR(Instruction Register，指令寄存器)`：当前正在执行的指令。
- `MAR(Memory Address Register，存储器地址寄存器)`：要访问的内存地址。
- `MDR(Memory Data Register，存储器数据寄存器)`：从内存读出或要写入内存的数据。
- `ACC(Accumulator，累加器)`：算术逻辑运算常用暂存区。
- `PSW(Program Status Word，程序状态字)`：标志位和状态信息。

#### 快速判断

- “下一条指令地址”一定找 `PC`。
- “当前指令内容”一定找 `IR`。
- “地址”和“数据”不要混，地址找 `MAR`，数据找 `MDR`。

#### 易错点

- `PC` 里放的是地址，不是指令本身。
- `IR` 里放的是指令，不是下一条地址。

### 2. 指令流水线(Pipeline，流水线) 计算题

#### 这题考什么

考流水线执行时间、吞吐率、加速比。

#### 这是什么

流水线本质上就是把“做一条指令”拆成几个小步骤，让多条指令像工厂流水线一样重叠执行。

直白地说：

- 不开流水线：第一条做完再做第二条
- 开流水线：第一条在执行时，第二条已经在取指，第三条可能已经在排队

所以流水线的核心不是“单条指令变快了”，而是“整体处理更多了”。

#### 最小概念

- 一条指令分成若干阶段，比如取指、分析、执行。
- 流水线的总时间本质是“首条指令走完整条流水线的时间 + 后面每条指令补一个节拍”。

#### 解题步骤

1. 先数阶段数 `k`。
2. 再数指令数 `n`。
3. 如果每段时间相同且为 `t`，流水线时间通常是 `(k + n - 1)t`。
4. 非流水线时间通常是 `nkt`。
5. 加速比 = 非流水线时间 / 流水线时间。
6. 吞吐率 = 完成指令数 / 总时间。

#### 微型例题

3 段流水线，执行 5 条指令，每段 2ns。

- 流水线时间 = `(3 + 5 - 1) × 2 = 14ns`
- 非流水线时间 = `5 × 3 × 2 = 30ns`
- 加速比 = `30 / 14`

#### 易错点

- 不要把流水线时间直接算成 `nkt`。
- 若题目给出每段时间不同，要取“最长段”为节拍，不再直接乘统一的 `t`。

### 3. CRC(Cyclic Redundancy Check，循环冗余校验) 计算校验码--

#### 这题考什么

考模 2 除法，求校验码或判断是否出错。

#### 这是什么

CRC 可以理解成“给数据再贴一个防伪标签”。

发送数据时，先按约定规则算出一小段校验码一起发出去；接收端再按同样规则验一遍，如果结果对不上，就说明传输过程中数据可能变了。

#### 最小概念

- 发送端：原始数据后补 `r` 个 0，`r` 是生成多项式位数减 1。
- 用模 2 除法去除，得到余数。
- 余数就是 CRC 校验码。

#### 解题步骤

1. 看生成多项式或除数有几位。
2. 在原数据后补对应数量的 0。
3. 用异或做模 2 除法，同位相同为 0，不同为 1，不借位。
4. 最后余数几位，就把它补回原数据后面。

#### 微型例题

原数据 `1011`，生成多项式 `1101`。

- 除数 4 位，所以补 3 个 0，得到 `1011000`
- 对 `1011000` 做模 2 除 `1101`
- 最终余数若为 `001`
- 发送码就是 `1011001`

#### 易错点

- 补 0 的个数不是看原数据长度，而是看除数位数减 1。
- 模 2 除法用异或，不是普通减法。

### 4. 海明码(Hamming Code，海明码) 位数题

#### 这题考什么

考校验位个数，或者总位数。

#### 这是什么

海明码也是一种“给原数据加检查信息”的方法，但它比普通奇偶校验更强，不只是能发现错，还常能定位哪一位错了。

考试里你先不用深究编码过程，最重要的是会算“最少要加几个校验位”。

#### 最小概念

如果信息位有 `m` 位，校验位有 `r` 位，则满足：

`2^r >= m + r + 1`

#### 解题步骤

1. 先看题目给的是信息位还是总位数。
2. 如果给信息位 `m`，试最小的 `r`，直到满足 `2^r >= m + r + 1`。
3. 总位数 = `m + r`。

#### 微型例题

信息位 4 位，求校验位。

- `r = 2` 时，`4 < 4 + 2 + 1 = 7`，不够
- `r = 3` 时，`8 >= 4 + 3 + 1 = 8`，够了
- 所以校验位 3 位，总位数 7 位

#### 易错点

- 不要背成 `2^r >= m + 1`，那是错的。
- 海明码常考“最少几个校验位”，所以要取最小满足值。

### 5. Cache(高速缓存) 映射--

#### 这题考什么

考直接映射、全相联映射、组相联映射的地址划分和命中分析。

#### 这是什么

Cache 就是 CPU 和内存之间的一层“小而快的临时仓库”。

因为内存比 CPU 慢，所以 CPU 会把常用数据先放到 Cache 里，下次再用时就能更快拿到。映射方式考的其实就是：主存里的某一块数据，允许放到 Cache 的哪里。

#### 最小概念

- `直接映射`：主存块只能放到 Cache 的唯一位置。
- `全相联映射`：主存块可以放到 Cache 任意块。
- `组相联映射`：主存块先定位到某组，再在组内任选位置。

#### 解题步骤

1. 先求主存块号。
2. 直接映射：`Cache 行号 = 主存块号 mod Cache 总行数`
3. 组相联：`组号 = 主存块号 mod 组数`
4. 地址通常拆成“标记位 + 组号或行号 + 块内地址”。
5. 判断命中时先看组或行，再看标记位是否一致。

#### 微型例题

主存共 32 块，Cache 共 8 行，采用直接映射。

- 主存块 13 映射到 `13 mod 8 = 5`
- 所以它只能去 Cache 第 5 行

#### 易错点

- “块号 mod 行数”算出来的是 Cache 行号，不是主存地址。
- 组相联不是对 Cache 总行数取模，而是对“组数”取模。

### 6. 冗余技术(Redundancy，冗余)

#### 这题考什么

考概念区分，常见于选择题。

#### 这是什么

冗余就是“故意多留一份保险”。

系统怕出错、怕坏、怕丢，所以会多做一次、多带一点校验信息，或者多准备一套设备。多出来的那部分，就是冗余。

#### 最小概念

- `时间冗余`：多执行几次，比对结果。
- `信息冗余`：附加校验位，比如 CRC、海明码。
- `结构冗余`：多放一套硬件，比如双机热备。

#### 速记

- 多算几次是时间冗余。
- 多加校验信息是信息冗余。
- 多备一套设备是结构冗余。

#### 易错点

- 校验码类题目本质属于信息冗余，不是结构冗余。

### 7. 关键路径(Critical Path，关键路径)

#### 这题考什么

考事件最早时间、最迟时间、时差、关键活动。

#### 这是什么

关键路径题本质是在问：一堆任务之间有先后依赖时，哪条任务链最拖工期。

那条“最拖时间”的路径就是关键路径。它上面任何一个活动晚了，整个项目就晚了。

#### 解题步骤

1. 顺推求每个事件最早发生时间。
2. 逆推求每个事件最迟发生时间。
3. 总时差 = 最迟开始时间 - 最早开始时间。
4. 时差为 0 的活动在关键路径上。
5. 关键路径长度就是项目最短工期。

#### 微型例题

若 `A(3天) -> B(2天) -> D(4天)`，另一条是 `A(3天) -> C(5天) -> D(4天)`。

- 路径 1 长度 = `3 + 2 + 4 = 9`
- 路径 2 长度 = `3 + 5 + 4 = 12`
- 关键路径是 `A-C-D`
- 最短工期 12 天

#### 易错点

- 最长路径才是关键路径，不是最短路径。
- 关键路径可能不止一条。

### 8. 子网划分(Subnetting，子网划分)

#### 这题考什么

考网络号、主机数量、掩码长度。

#### 这是什么

子网划分就是把一个大网段切成多个小网段，方便管理、隔离广播和节省地址。

考试里你可以把它理解成“拿 IP 地址做分组”，核心就是判断：

- 这一段网络从哪开始
- 到哪结束
- 能放多少台主机

#### 最小概念

- 前缀长度 `/n` 表示前 `n` 位是网络位。
- 主机位数 = `32 - n`
- 可用主机数通常是 `2^(主机位数) - 2`

#### 解题步骤

1. 看掩码前缀，比如 `/27`。
2. 主机位 = `32 - 27 = 5`
3. 每个子网地址数 = `2^5 = 32`
4. 可用主机数 = `32 - 2 = 30`
5. 子网跨度 = 32，所以网络号按 32 递增

#### 微型例题

`192.168.1.64/27`

- 每段跨度 32
- 网络号可能是 `0、32、64、96...`
- 所以该地址所在子网网络号是 `192.168.1.64`
- 广播地址是 `192.168.1.95`
- 可用主机范围 `65` 到 `94`

#### 易错点

- 可用主机数一般减 2，要排除网络号和广播地址。
- 题目若明确是点对点链路，可能不按减 2 处理，要看题意。

### 9. PV 操作--

#### 这题考什么

考信号量(Semaphore，信号量) 的同步与互斥。

#### 这是什么

PV 操作本质上是在解决“多个进程一起干活时，怎么别打架、别乱顺序”。

- 互斥：同一时刻谁能进去
- 同步：谁必须等谁做完才能继续

你可以把信号量理解成一个“带数字的门禁牌”。

#### 最小概念

- `P` 操作：申请资源，信号量减 1。
- `V` 操作：释放资源，信号量加 1。
- `互斥`：同一时刻只能一个进程进入临界区。
- `同步`：进程间按顺序配合执行。

#### 解题套路

1. 先判断题目是在考互斥还是同步。
2. 互斥题：通常设置一个初值为 1 的信号量。
3. 同步题：前一事件完成后 `V`，后一事件开始前 `P`。
4. 看到“缓冲区空位数”“产品数”，往往是生产者消费者模型。

#### 微型例题

单缓冲区生产者消费者：

- `empty = 1`
- `full = 0`
- `mutex = 1`

生产者顺序：

`P(empty) -> P(mutex) -> 放产品 -> V(mutex) -> V(full)`

消费者顺序：

`P(full) -> P(mutex) -> 取产品 -> V(mutex) -> V(empty)`

#### 易错点

- 同步和互斥的信号量不要混用。
- `P/V` 顺序乱了会导致死锁或越界。

### 10. 资源图、死锁、银行家算法(Banker's Algorithm，银行家算法)--

#### 这题考什么

考死锁条件判断、安全序列判断。

#### 这是什么

这类题在研究“资源不够时，多个进程会不会互相卡死”。

- 死锁：大家都等别人手里的资源，结果谁也走不了
- 银行家算法：像银行放贷一样，分资源前先算一遍，确认分出去后系统不会陷入危险

#### 最小概念

- 死锁四条件：互斥、请求并保持、不剥夺、环路等待。
- `资源分配图`里若只有单实例资源，出现环可能死锁。
- 银行家算法核心是：分配后系统还能不能找到一个安全序列。

#### 银行家算法步骤

1. 算 `Need = Max - Allocation`
2. 初始 `Work = Available`
3. 找一个进程满足 `Need <= Work`
4. 让它完成，释放资源：`Work = Work + Allocation`
5. 重复直到所有进程都能完成，得到安全序列

#### 微型例题

若当前 `Work = (3,3,2)`，某进程 `Need = (1,2,2)`，则它可执行。

执行完若其 `Allocation = (1,0,1)`：

- 新 `Work = (3,3,2) + (1,0,1) = (4,3,3)`

#### 易错点

- 银行家算法不是看“当前够不够分”，而是看“分完后是否仍安全”。
- 资源图出现环时，若资源有多实例，不一定死锁。

### 11. 分页(Paging，分页) / 分段(Segmentation，分段) / 段页式

#### 这题考什么

考概念区分和地址转换。

#### 这是什么

这几个概念都在回答同一个问题：程序装进内存后，操作系统怎么管理它。

- 分页：站在机器管理角度，按固定大小切块
- 分段：站在程序逻辑角度，按功能模块切块
- 段页式：先按逻辑分，再按固定大小分

#### 最小概念

| 方式 | 单位 | 地址结构 | 主要特点 |
| --- | --- | --- | --- |
| 分页 | 页 | 页号 + 页内位移 | 方便离散存储，无逻辑意义 |
| 分段 | 段 | 段号 + 段内位移 | 按逻辑模块划分，便于共享保护 |
| 段页式 | 先段后页 | 段号 + 页号 + 页内位移 | 兼顾逻辑性和离散分配 |

#### 快速判断

- “逻辑模块、过程、数组、数据段”更像分段。
- “固定大小、页表、页号”更像分页。
- 题目同时出现段表和页表，就是段页式。

#### 微型例题

逻辑地址如果写成 `(段号, 页号, 页内位移)`，一定是段页式。

#### 易错点

- 分页是物理管理更友好，分段是程序员视角更自然。
- 页大小固定，段大小通常不固定。

### 12. 页面替换

#### 这题考什么

考最近未使用类算法，特别是“先看未访问，再看未修改”的分类。

#### 这是什么

内存放不下所有页面时，操作系统得决定“把谁先赶出去”。页面替换题考的就是这个决策规则。

如果一个页最近没用过，而且也没改过，那通常最适合先换出去，因为代价最小。

#### 最小概念

常见按访问位和修改位分 4 类：

- `00`：未访问，未修改，优先淘汰
- `01`：未访问，已修改
- `10`：已访问，未修改
- `11`：已访问，已修改，最后淘汰

#### 解题步骤

1. 先看访问位 `R`。
2. 优先找 `R=0` 的页。
3. 如果还要比较修改位 `M`，优先 `M=0`。
4. 淘汰顺序通常是 `00 -> 01 -> 10 -> 11`

#### 易错点

- 不是先看修改位，而是先看访问位。
- 已修改页换出时通常需要回写，代价更大，所以一般后淘汰。

### 13. Bitmap(位示图)

#### 这题考什么

考磁盘块或内存块的分配状态表示。

#### 这是什么

位示图就是拿一串 0 和 1 去记录“哪些块已经被占了，哪些块还是空的”。

它很像停车场的占位牌：

- 一个格子对应一个块
- 一个二进制位表示这个块现在的状态

#### 最小概念

- 一个二进制位对应一个块。
- `0/1` 分别表示空闲或占用，具体看题目定义。

#### 解题套路

1. 先确认 `0` 和 `1` 各表示什么。
2. 找连续空闲块时，从位示图中找连续相同位。
3. 求块号时，按位序换算。

#### 微型例题

若某位示图中第 5 到第 8 位都是 0，且题目规定 0 表示空闲，则存在 4 个连续空闲块。

#### 易错点

- 不同题目里 `0` 和 `1` 的含义可能相反，先看题干。

### 14. 索引文件

#### 这题考什么

考直接索引、一级间接索引能表示多大文件，或者访问某块需要几次读盘。

#### 这是什么

索引文件可以理解成“文件块地址目录”。

文件太大时，系统不可能只靠一个连续空间来存，所以会记录“这个文件的第几块数据放在哪里”。直接索引是目录直接指向数据，一级间接索引是目录先指到一个“二级目录页”，再由它指向数据。

#### 最小概念

- `直接索引`：索引表里直接存放数据块地址。
- `一级间接索引`：先找到一级索引块，再从索引块中找到数据块地址。

#### 解题步骤

1. 看一个盘块大小 `B`。
2. 看一个地址项大小 `A`。
3. 一级索引块最多能存 `B / A` 个地址项。
4. 直接索引容量 = 直接地址项数 × `B`
5. 一级间接索引容量 = `(B / A) × B`

#### 微型例题

盘块大小 1KB，地址项 4B，则一个一级索引块可存：

`1024 / 4 = 256` 个地址

若每个地址指向一个 1KB 数据块，则一级间接可表示：

`256 × 1KB = 256KB`

#### 易错点

- 一级间接索引要多经过一次索引块，访问次数比直接索引多。
- 不要把“地址项个数”和“数据块大小”混起来。

### 15. ER 图(Entity Relationship Diagram，实体联系图)

#### 这题考什么

考实体、属性、联系，尤其是 `1:1`、`1:N`、`M:N` 转关系模式。

#### 这是什么

ER 图就是数据库建表前的“业务草图”。

它先不急着写 SQL，而是先把业务世界里的东西画清楚：

- 有哪些对象
- 每个对象有什么信息
- 对象之间是什么关系

#### 最小概念

- `实体`：客观对象，如学生、课程。
- `属性`：实体特征，如学号、姓名。
- `联系`：实体之间的关系，如选课。

#### ER 转关系模式口诀

- 实体变表。
- 主键保留。
- `1:1` 联系可把外键放任一方，优先放“参与全部”的一方。
- `1:N` 联系把 `1` 方主键放到 `N` 方当外键。
- `M:N` 联系单独拆中间表，中间表至少含两边主键。

#### 微型例题

学生和课程是 `M:N` 的“选修”关系。

应拆成：

- 学生表
- 课程表
- 选修表(`学号`, `课程号`, 其他属性)

#### 易错点

- `M:N` 不能直接互相放外键，必须拆中间表。

### 16. 范式(Normal Form，规范化)

#### 这题考什么

考 1NF、2NF、3NF 判断和优化。

#### 这是什么

范式本质上是在解决“表设计得乱不乱、会不会重复、会不会出更新问题”。

你可以把它理解成数据库表的整理规则，目的是减少冗余、避免改一处漏一处。

#### 最小概念

- `1NF`：属性不可再分。
- `2NF`：在 1NF 基础上，非主属性完全依赖主键，解决部分函数依赖。
- `3NF`：在 2NF 基础上，非主属性不传递依赖主键。

#### 快速判断口诀

- 看字段是不是原子值，先判 1NF。
- 看联合主键下，某字段是否只依赖主键一部分，判 2NF。
- 看非主属性是否依赖另一个非主属性，判 3NF。

#### 微型例题

关系 `选课(学号, 课程号, 学生姓名, 成绩)`，主键是 `(学号, 课程号)`。

- `学生姓名` 只依赖 `学号`
- 所以存在部分依赖
- 不是 2NF

#### 易错点

- 判断 2NF 时，必须先看主键是不是联合主键。
- 单属性主键的表，通常不会有部分函数依赖问题。

### 17. 数据流图 DFD(Data Flow Diagram，数据流图) 与数据字典

#### 这题考什么

考补图、改错、平衡检查。

#### 这是什么

DFD 不是画类，不是画数据库，也不是画代码流程。它画的是“数据在系统里怎么流动”。

也就是说，它关心的是：

- 数据从谁来
- 到系统里经过哪些处理
- 存到哪里
- 最后给谁

#### 最小概念

DFD 的基本元素：

- 外部实体
- 加工
- 数据存储
- 数据流

数据字典(Data Dictionary，数据字典) 是对数据流、数据项、存储、加工的文字定义。

#### DFD 做题顺序

1. 先找题干中的“人、系统、部门”，补外部实体。
2. 再找“动作”，补加工。
3. 再找“长期保存的数据”，补数据存储。
4. 最后连数据流，名字要是数据，不要是动作。
5. 做平衡检查：父图出现的输入输出，子图不能凭空多也不能凭空少。

#### 常见错误

- 数据流名称写成“查询”“修改”这种动词。
- 外部实体直接连数据存储。
- 数据存储之间直接相连。
- 子图比父图多出一个外部输入。

#### 易错点

- 加工可以输入输出数据流，但外部实体和数据存储不能随便直连。

## 软件工程与设计

### 18. 内聚(Cohesion，内聚) 和耦合(Coupling，耦合)

#### 这题考什么

考概念排序和判断。

#### 这是什么

这两个词本质上是在评价设计好不好。

- 内聚看“一个模块内部是不是只干一类事”
- 耦合看“模块之间是不是缠得太死”

所以它们一个看“内部”，一个看“外部”。

#### 最小概念

- `内聚`：一个模块内部各成分联系有多紧密，越高越好。
- `耦合`：模块之间依赖有多强，越低越好。

#### 常见排序

内聚从低到高大致记：

偶然内聚 < 逻辑内聚 < 时间内聚 < 过程内聚 < 通信内聚 < 顺序内聚 < 功能内聚

耦合从低到高大致记：

非直接耦合 < 数据耦合 < 标记耦合 < 控制耦合 < 外部耦合 < 公共耦合 < 内容耦合

#### 速记

- 好设计：高内聚、低耦合。
- “一个模块只做一件事”是在追求功能内聚。

#### 易错点

- 不要把“高耦合”理解成“协作更强所以更好”，考试里恰好相反。

### 19. CMMI(Capability Maturity Model Integration，能力成熟度模型集成) 五级

#### 这题考什么

考 5 个等级名称和含义。

#### 这是什么

CMMI 说白了是在评价一个软件组织“做项目有多成熟、多规范”。

等级越高，不代表代码一定更强，而是说明这个组织在过程管理、度量和持续改进上更成熟。

#### 速记表

| 等级 | 名称 | 记忆点 |
| --- | --- | --- |
| 1 | 初始级 | 靠个人，过程混乱 |
| 2 | 已管理级 | 项目级可管理 |
| 3 | 已定义级 | 组织级标准化 |
| 4 | 定量管理级 | 用数据度量和控制 |
| 5 | 优化级 | 持续改进 |

#### 速记口诀

初始、已管、已定义、定量、优化。

#### 易错点

- 2 级是“已管理”，3 级才是“已定义”。

### 20. UML(Unified Modeling Language，统一建模语言) 高频点

#### 这题考什么

考类图、用例图、时序图、活动图。

#### 这是什么

UML 是一套“画软件设计图的通用语言”。

不同图回答的问题不同：

- 类图：系统里有什么类，它们怎么关联
- 用例图：系统给谁提供什么功能
- 时序图：对象之间先后怎么交互
- 活动图：流程怎么走

#### 类图速记

- `泛化`：继承，空心三角。
- `依赖`：临时使用，虚线箭头。
- `关联`：长期知道对方，实线。
- `聚合`：整体可离开部分，空心菱形。
- `组合`：整体消失部分也没了，实心菱形。

#### 用例图速记

- 参与者在系统外。
- 用例在系统边界内。
- `include` 表示必然复用。
- `extend` 表示可选扩展。

#### 时序图速记

- 先看对象生命线。
- 再看消息顺序。
- 时间从上往下。

#### 活动图速记

- 更像流程图，关注活动流转和分支汇合。

#### 易错点

- 聚合和组合最容易混。
- `include` 是一定发生，`extend` 是有条件才发生。

### 21. 设计模式(Design Pattern，设计模式) 高频识别

#### 这题考什么

考场景选模式、补类图、补代码。

#### 这是什么

设计模式不是死规定，也不是语法点，它更像“别人总结好的常见解法模板”。

考试里其实不是让你背故事，而是看见场景后能判断：这种变化和协作关系，最适合套哪种老练做法。

#### 高频模式一览

| 模式 | 什么时候用 | 一句话记忆 |
| --- | --- | --- |
| 单例模式 | 全局只有一个对象 | 构造私有，自己管自己 |
| 工厂模式 | 创建逻辑想和使用分离 | 把 new 藏起来 |
| 观察者模式 | 一处变化，多处通知 | 发布订阅 |
| 策略模式 | 同一任务，多种算法切换 | 换算法不换调用方 |
| 适配器模式 | 旧接口不兼容新需求 | 包一层翻译 |
| 桥接模式 | 两个维度都可能变化 | 抽象和实现分离 |
| 模板方法模式 | 主流程固定，步骤可改 | 父类定骨架，子类填步骤 |

#### 看到关键词怎么想

- “全局唯一”想到单例模式。
- “不同支付方式、不同排序策略”想到策略模式。
- “兼容老接口”想到适配器模式。
- “通知多个界面更新”想到观察者模式。
- “流程固定但步骤不同”想到模板方法模式。

#### 易错点

- 工厂模式解决“创建对象”的变化。
- 策略模式解决“算法行为”的变化。
- 适配器不是新增功能，本质是兼容接口。

### 22. 非对称加密(Asymmetric Encryption，非对称加密)

#### 这题考什么

考公钥、私钥在加密和签名中的用法。

#### 这是什么

非对称加密就是一对钥匙分工合作：

- 一把公开给别人，叫公钥
- 一把自己保管，叫私钥

它解决两个不同问题：保密和证明身份。

#### 最小概念

- `公钥加密，私钥解密`：用于保密。
- `私钥签名，公钥验证`：用于认证和防抵赖。

#### 速记

- 想保密：用对方公钥加密，只有对方私钥能解。
- 想证明“就是我发的”：用自己私钥签名，别人拿我公钥验证。

#### 易错点

- “公钥签名”是错的。
- “私钥加密用于保密”通常不是考试标准说法，保密考的是公钥加密。

### 23. 常见网络协议

#### 这是什么

网络协议你可以理解成“不同网络服务约定好的说话方式”。

邮件、网页、文件传输、域名解析，这些事情都不是乱传数据，而是按各自协议在通信。考试常考的是“哪个协议是干什么的”。

#### 邮件协议

| 协议 | 作用 | 默认端口 | 记忆点 |
| --- | --- | --- | --- |
| SMTP(Simple Mail Transfer Protocol，简单邮件传输协议) | 发邮件 | 25 | 发送 |
| POP3(Post Office Protocol 3，邮局协议第三版) | 收邮件 | 110 | 下载到本地 |
| IMAP(Internet Message Access Protocol，互联网邮件访问协议) | 收邮件 | 143 | 邮件保留在服务器 |

#### 文件传输

| 协议 | 作用 | 记忆点 |
| --- | --- | --- |
| FTP(File Transfer Protocol，文件传输协议) | 可靠文件传输 | 功能完整 |
| TFTP(Trivial File Transfer Protocol，简单文件传输协议) | 简化版文件传输 | 简单、轻量 |

#### 域名和安全

| 协议 | 作用 | 记忆点 |
| --- | --- | --- |
| DNS(Domain Name System，域名系统) | 域名解析 | 域名转 IP |
| HTTP(HyperText Transfer Protocol，超文本传输协议) | 普通网页访问 | 明文 |
| HTTPS(HyperText Transfer Protocol Secure，安全超文本传输协议) | 加密网页访问 | HTTP + TLS |

#### 易错点

- 发邮件是 `SMTP`，收邮件是 `POP3/IMAP`。
- `POP3` 更偏“下载走”，`IMAP` 更偏“服务器同步”。

### 24. DFD 大题模板

#### 这是什么题

这类题不是让你发明业务，而是让你根据题干把“数据流动图”补完整、改正确。

#### 拿题先做什么

1. 读题干中的参与方，圈出外部实体。
2. 读题干中的业务动作，圈出加工。
3. 读题干中的持久数据，圈出数据存储。
4. 再去补数据流名称，统一写成名词。

#### 平衡检查模板

1. 看父图的输入有没有在子图出现。
2. 看父图的输出有没有在子图出现。
3. 子图不能平白新增父图没有的外部输入输出。

#### 改错高频点

- 外部实体不能直接访问数据存储。
- 数据流名字不要写动词。
- 编号层次要合理，父加工分解成子加工。

### 25. 数据库设计大题模板

#### 这是什么题

这类题本质是“把业务描述翻译成数据库结构”，通常会经历：找实体、判关系、转表、查范式。

#### 拿题先做什么

1. 先找实体。
2. 再找每个实体的主键。
3. 再看实体间基数，是 `1:1`、`1:N` 还是 `M:N`。
4. `M:N` 优先考虑拆中间表。
5. 最后再检查范式问题。

#### ER 转表模板

- 每个实体一张表。
- 主键保留。
- `1:N`：`1` 方主键去 `N` 方。
- `M:N`：单独建关系表。
- 多值属性通常单独拆表。

#### SQL 常考思路

- 查找类：先看是连接、分组还是子查询。
- 统计类：重点关注 `COUNT`、`SUM`、`AVG`、`GROUP BY`。
- 条件类：先写 `WHERE`，再想是否需要聚合过滤 `HAVING`。

#### 事务(Transaction，事务) 题速记

- 原子性：要么全做，要么全不做。
- 一致性：前后都满足约束。
- 隔离性：互不干扰。
- 持久性：提交后不丢。

### 26. UML 大题模板

#### 这是什么题

这类题是在考你能不能把文字需求翻译成图上的关系和交互，而不是考你背定义。

#### 类图题先看什么

1. 看有没有“是一个”关系，有就先想泛化。
2. 看有没有“整体-部分”关系，再区分聚合还是组合。
3. 看对象之间是临时调用还是长期持有，区分依赖和关联。

#### 用例图题先看什么

1. 谁在系统外，就画参与者。
2. 谁是系统提供的功能，就画用例。
3. 公共必做步骤用 `include`。
4. 有条件的附加功能用 `extend`。

#### 时序图题先看什么

1. 谁先发起消息。
2. 哪个对象负责主要控制。
3. 返回消息和创建销毁点是否需要体现。

### 27. 设计模式大题模板

#### 这是什么题

这类题是在考“你能不能识别变化点，并选出一个合适的结构去承载这种变化”。

#### 拿到题先问自己 3 个问题

1. 变化点在哪里，是对象创建变了，还是算法变了，还是接口不兼容？
2. 客户端想不想感知这种变化？
3. 有没有“一处变化，多处联动通知”的需求？

#### 题干到模式的映射

- 多种算法可替换：策略模式。
- 老系统接口不兼容：适配器模式。
- 只允许一个实例：单例模式。
- 对象创建过程复杂且想解耦：工厂模式。
- 一个对象变化通知多个对象：观察者模式。
- 抽象和实现两个维度独立变化：桥接模式。
- 流程骨架不变，局部步骤可定制：模板方法模式。

### 28. 算法设计大题模板

#### 这是什么题

这类题不只是写代码，它通常是在考你识别问题类型、补全关键逻辑、分析复杂度。

#### 第一步不是写代码，而是分类

| 看到什么信号 | 先想什么方法 |
| --- | --- |
| 分成左右两半，递归合并 | 分治 |
| 枚举所有可能，走不通就回退 | 回溯 |
| 最优子结构，重复子问题 | 动态规划 |
| 每步先取当前最好 | 贪心 |
| 在图或状态空间里找路径 | 搜索 |

#### 常见填空位

- 递归出口有没有写。
- 状态转移方程有没有漏边界。
- 回溯时有没有“撤销选择”。
- 双重循环顺序是否正确。
- 数组下标是否越界。

#### 复杂度分析速记

- 单层循环一般是 `O(n)`。
- 双层嵌套一般是 `O(n^2)`。
- 每次规模减半常见 `O(logn)`。
- 归并排序是 `O(nlogn)`。
- 快速排序平均 `O(nlogn)`，最坏 `O(n^2)`。

#### C 语言风格答题提醒

- 先补函数定义和参数含义。
- 再补递归出口或循环边界。
- 最后检查返回值和数组下标。

### 29. C++ 面向对象大题模板

#### 这是什么题

这类题本质是在考你有没有真正理解“对象怎么协作、继承怎么复用、多态怎么生效”。

#### 考前要稳住的点

- 继承：子类复用父类。
- 多态(Polymorphism，多态)：父类指针指向子类对象，调用重写后的函数。
- 虚函数(`virtual`)：实现运行时多态的关键。
- 纯虚函数：`virtual void f() = 0;`，含纯虚函数的类是抽象类，不能实例化。
- 异常处理：`try-catch-throw`。

#### 拿题先看什么

1. 有没有父类指针或引用指向子类对象。
2. 基类函数是否声明为 `virtual`。
3. 是否在考构造析构顺序。
4. 是否在考模式和面向对象结合。

#### 常见模式结合点

- 单例模式：私有构造 + 静态实例。
- 工厂模式：返回父类指针，实际创建不同子类对象。
- 策略模式：上下文持有策略接口指针。
- 观察者模式：主题维护观察者列表并通知。

#### 易错点

- 基类析构函数在多态场景最好写成虚析构函数。
- 没有 `virtual`，父类指针调用的还是父类版本。

### 易混对照表
